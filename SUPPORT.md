# Support

## Start with the repo docs
- `README.md` for project overview and run instructions
- `CURRENT_STATE.md` for architecture and phase-status truth
- `RELEASE_NOTES.md` for the current shippable claim
- `lcars-ui/docs/TARGET_BANK_ACCEPTANCE.md` for canonical acceptance scope

## Use the right GitHub channel
- Reproducible bug or regression: open a GitHub issue with environment details, exact commands, expected behavior, actual behavior, and a minimal repro when possible.
- Usage question: use GitHub Discussions if enabled; otherwise open an issue and prefix the title with `question:`.
- Security-sensitive report: do not use a public issue; follow `SECURITY.md`.
- Release/publication question: reference `RELEASE_NOTES.md` and `GITHUB_PUBLICATION_CHECKLIST.md` so discussion stays tied to the published claim.
